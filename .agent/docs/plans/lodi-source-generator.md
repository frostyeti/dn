# Lodi Source Generator Plan

## Goal

Eliminate reflection from FrostYeti.Lodi by adding strongly-typed APIs and a source generator for compile-time DI registration.

## Phase 1: API Improvements (No Generator)

**Files to modify:**
- `bcl/Lodi/src/LodiServiceProvider.cs`

**New methods:**

```csharp
// Strongly-typed registration with implementation type
RegisterTransient<TService, TImpl>(Func<LodiServiceProvider, TImpl> factory)
RegisterScoped<TService, TImpl>(Func<LodiServiceProvider, TImpl> factory)
RegisterSingleton<TService, TImpl>(Func<LodiServiceProvider, TImpl> factory)
RegisterSingletonInstance<TService, TImpl>(TImpl instance) where TImpl : TService

// Try-register pattern (returns false if already registered)
TryRegisterTransient<T>(Func<LodiServiceProvider, T> factory) -> bool
TryRegisterScoped<T>(Func<LodiServiceProvider, T> factory) -> bool
TryRegisterSingleton<T>(Func<LodiServiceProvider, T> factory) -> bool
TryRegisterSingletonInstance<T>(T instance) -> bool

// Inspection
bool ContainsService<T>()
bool ContainsService(string key)
```

## Phase 3: Attributes

**File:** `bcl/Lodi/src/LodiAttributes.cs`

```csharp
namespace FrostYeti.Lodi;

/// <summary>
/// Marks a class for automatic DI registration via source generator.
/// </summary>
[AttributeUsage(AttributeTargets.Class, AllowMultiple = false, Inherited = false)]
public sealed class LodiServiceAttribute : Attribute
{
    /// <summary>
    /// Gets or sets the service lifetime.
    /// </summary>
    public ServiceLifetime Lifetime { get; set; } = ServiceLifetime.Transient;

    /// <summary>
    /// Gets or sets the service type (interface) to register as.
    /// If null, registers as the implementation type.
    /// </summary>
    public Type? ServiceType { get; set; }

    /// <summary>
    /// Gets or sets the key for named registration.
    /// </summary>
    public string? Key { get; set; }
}

/// <summary>
/// Allows multiple service registrations for a single implementation.
/// </summary>
[AttributeUsage(AttributeTargets.Class, AllowMultiple = true, Inherited = false)]
public sealed class LodiServiceImplementationAttribute : Attribute
{
    public LodiServiceImplementationAttribute(Type serviceType)
    {
        ServiceType = serviceType;
    }

    public Type ServiceType { get; }
    public ServiceLifetime Lifetime { get; set; } = ServiceLifetime.Transient;
    public string? Key { get; set; }
}
```

## Phase 4: Generator Output Examples

### Input

```csharp
[LodiService(ServiceType = typeof(IUserService), Lifetime = ServiceLifetime.Scoped)]
public class UserService(IUserRepository repo, ILogger<UserService> logger) : IUserService
{
}
```

### Generated

```csharp
// <auto-generated />
namespace FrostYeti.Lodi.Generated;

public static partial class LodiServiceExtensions
{
    public static LodiServiceProvider AddUserService(this LodiServiceProvider provider)
    {
        return provider.RegisterScoped<IUserService>(sp =>
            new UserService(
                sp.GetRequiredService<IUserRepository>(),
                sp.GetRequiredService<ILogger<UserService>>()));

    }

    public static bool TryAddUserService(this LodiServiceProvider provider)
    {
        if (provider.ContainsService<IUserService>())
            return false;

        provider.RegisterScoped<IUserService>(sp =>
            new UserService(
                sp.GetRequiredService<IUserRepository>(),
                sp.GetRequiredService<ILogger<UserService>>()));
        return true;
    }
}
```

## Phase 5: Generator Implementation

### 5.1 LodiServiceGenerator

1. Find types with `[LodiService]` attribute
2. Analyze primary constructor (or single constructor)
3. Resolve parameter types from constructor
4. Generate `Add{TypeName}` extension method
5. Generate `TryAdd{TypeName}` extension method
6. Handle named services via `Key` property

### 5.2 LodiModuleGenerator

1. Find all `[LodiService]` types in assembly
2. Generate `AddServicesFromAssembly(this LodiServiceProvider)` method
3. Optionally support module grouping via `[LodiModule]` attribute

## Phase 6: Compile-Time Diagnostics

Generator produces diagnostics for:

| Code | Severity | Description |
|------|----------|-------------|
| LDI001 | Warning | Service depends on unregistered type |
| LDI002 | Error | Singleton depends on Scoped service |
| LDI003 | Error | Circular dependency detected |
| LDI004 | Warning | Multiple constructors - specify which to use |
| LDI005 | Error | No suitable constructor found |
